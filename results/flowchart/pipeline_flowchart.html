IL-10-Induced Peptide Model — 訓練 Pipeline

入口 main.py → src/pipeline.py · run_pipeline()　·　三個模型 variant（共用 ESM 嵌入後分流）　·　評估只用 train OOF 學門檻

1 · 入口

CLI 參數 → PipelineConfig main.py

解析參數後建立 PipelineConfig 並呼叫 run_pipeline()。

  • --variant esm_only / handcrafted_no_fs / handcrafted_with_fs / all
  • --esm-model 預設 esm2_t33_650M_UR50D、--batch-size 32、--num-folds 5、--seed 42
  • 輸入：data/peptide_level_dataset_MHCII.csv（train）、data/all_minus_benchmark_minus_mhcii.csv（independent test）
2 · 資料處理

載入與解析序列 src/data.py · load_datasets()

  • 解析每筆 Sequence：拆成 BaseSeq + Annotation，偵測修飾關鍵字 PHOS/DEAM/CITR/ACET/MCM
  • 產生中介特徵：HasMod / ModCount / SeqLength / LogLength 等 modification metadata
  • 只保留標準 20 種胺基酸序列（IsStandardBaseSeq 過濾）
  • 輸出稽核 results/data_audit.json（正負比例、train/test 序列重疊度等）
3 · 共用嵌入

ESM-2 序列嵌入（所有 variant 共用）src/features.py · ESMEmbedder

對 BaseSeq 跑 ESM-2，取每個殘基表示後做 mean pooling 與 max pooling，結果快取到 artifacts/cache/*.npz（自動偵測 CUDA / CPU）。

🔀 依 variant 分流（--variant all 時三條都跑，逐一進入 _run_single_variant()）
esm_only

ESM-only Stacking

  1. 組三種 view：mean / max / concat，各自再接上 modification metadata
  2. 掃 3 組 tuning candidate（mean+max、+concat、regularized）
  3. 每組做巢狀 CV stacking：每個 view 一個 XGBoost 當 base，再用 Logistic Regression 當 meta-model 疊合
  4. 用 train OOF 的 PR-AUC / MCC 選出最佳 candidate → tuning_summary.csv
train_esm_stacking_variant()
handcrafted_no_fs

ESM + 手工特徵（不做篩選）

  1. 建手工特徵 AAC + DPC + aaIndex + mod（build_handcrafted_features）
  2. 與 ESM mean+max embedding 串接成單一特徵矩陣
  3. 不做任何特徵篩選
  4. 單一 XGBoost：StratifiedKFold 產生 OOF，再用平均最佳迭代數重訓全資料模型
train_single_xgb_variant()
handcrafted_with_fs

ESM + 手工特徵（含篩選）

  1. 同上組合 ESM + AAC + DPC + aaIndex + mod
  2. 三段特徵篩選：① 變異數過濾 ② aaIndex 相關性過濾（|r|>0.95）③ XGBoost 重要度過濾 → feature_selection_summary.json
  3. 篩選後特徵丟進同一個單一 XGBoost
  4. StratifiedKFold OOF + 全資料重訓
train_single_xgb_variant()
三條分支都產出 train OOF 機率 與 independent test 機率，匯入相同評估流程
5 · 評估與門檻

指標與決策門檻 src/model.py · build_metric_tables()

  • 門檻只從 train OOF 機率學：default 0.5、best MCC、best F2
  • 對 TrainOOF 與 IndependentTest 各算：ROC-AUC、PR-AUC、MCC、Balanced Acc、Precision、Recall/Sensitivity、Specificity、F1、F2、混淆矩陣

寫入 results/<variant>/ 與 artifacts/models/

7 · 跨 variant 比較

彙整比較表 _write_comparison()

取各 variant 在 IndependentTest × best-MCC 門檻 的成績，依 MCC、PR-AUC 排序 → results/model_comparison_summary.csv 與 model_comparison_full_metrics.csv。
